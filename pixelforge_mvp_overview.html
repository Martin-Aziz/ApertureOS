Canvas Engine

WebGL2 + WebAssembly
GPU-accelerated compositing. Rust compiled to WASM for pixel ops. 60fps on 100MP images.

Frontend

SvelteKit + TypeScript
Fastest DOM diffing for real-time canvas. Smaller bundle than React for editor-class apps.

Backend API

Rust (Axum)
File upload, PSD parsing, project CRUD, auth. Memory-safe, zero-cost abstractions.

AI Service

Python FastAPI + REMBG + ONNX
Background removal, upscaling. Separate service, horizontally scalable.

Database

PostgreSQL + S3/R2
Project metadata in PG, large files (PSD, exports) in object storage with CDN.

Auth

Supabase Auth
Email, GitHub OAuth, magic link. JWT access tokens, 15-min expiry, refresh rotation.

Collab (V1)

Yjs CRDTs + WebSocket
Conflict-free replicated data types. Industry standard (Figma uses similar). Partykit for hosting.

Error tracking, perf tracing, structured logs. Open source stack.

Canvas performance

WebGL2 on large images (100MP+) is hard. Mitigation: tile-based rendering, GPU memory budgets, WASM pixel ops, progressive render.

PSD fidelity

Adobe's PSD spec has undocumented features. Use ag-psd + fallback to rasterizing unsupported layer types. Be transparent with users.

AI service cost at scale

GPU inference is expensive. Mitigation: client-side WASM model for background removal in V1, rate-limit AI calls for free tier.

Community trust / adoption

Designers are skeptical of new tools. Mitigation: open source from day 1, public roadmap, Figma/GIMP import, strong docs.

Undo/redo complexity

Photoshop-class undo history with large raster data is memory-intensive. Use diff-based command pattern + step snapshots.

Licensing (AGPL/MIT)

Choose MIT for maximum adoption. AGPL forces SaaS forks to contribute back but deters enterprise. Recommendation: MIT core + AGPLv3 cloud sync.
